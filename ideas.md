<idea>
  **Design Movement**: Cyber-Noir Minimalism
  **Core Principles**:
  1. **Immersive Darkness**: Utilizing deep blacks and charcoal grays to create a premium, tech-forward atmosphere that feels infinite and focused.
  2. **Neon Precision**: Using sharp, glowing accents (blue, green, yellow) to guide attention and indicate status, mimicking a heads-up display (HUD).
  3. **Fluid Progression**: The form should feel like a conversation with an AI, not a static document. Smooth transitions and progressive disclosure are key.
  4. **Data-First Clarity**: Information density is managed through clear hierarchy and distinct visual grouping, making complex data entry feel effortless.

  **Color Philosophy**:
  - **Deep Void (#0a0a0a)**: Represents the depth and capability of the AI.
  - **Electric Blue (#3B82F6)**: Symbolizes intelligence and voice processing (active states).
  - **Signal Green (#22C55E)**: Indicates success, completion, and "go" signals.
  - **Warning Yellow**: Used sparingly for critical alerts or trial-related information.
  - The dark theme reduces eye strain and aligns with the "always-on" nature of AI services.

  **Layout Paradigm**:
  - **Split-Screen Asymmetry**: A fixed left panel with progress tracking and context, while the right panel handles the dynamic form inputs. This anchors the user while allowing for fluid movement through questions.
  - **Card-Based Input**: Each question or group of questions lives in a focused "card" that slides into view, reducing cognitive load.

  **Signature Elements**:
  - **Glowing Borders**: Subtle, pulsing borders on active input fields to mimic AI "listening" or "processing".
  - **Monospace Accents**: Using a monospace font for labels, numbers, and technical details to reinforce the "code/tech" aesthetic.
  - **Glassmorphism**: Subtle frosted glass effects on the progress sidebar to add depth without clutter.

  **Interaction Philosophy**:
  - **Instant Feedback**: Every input should trigger a subtle visual confirmation (e.g., a checkmark appearing, a border changing color).
  - **Keyboard Efficiency**: The form should be fully navigable via keyboard, with clear focus states.
  - **Progressive Disclosure**: Only show relevant follow-up questions based on previous answers (e.g., CRM details only appear if a CRM is selected).

  **Animation**:
  - **Slide & Fade**: Questions slide in from the bottom and fade out to the top, creating a vertical flow.
  - **Staggered Reveal**: Options within a question appear sequentially (fast stagger) to feel dynamic.
  - **Pulse Effects**: Subtle pulsing on the "Next" button when a section is complete.

  **Typography System**:
  - **Headings**: **Inter** (Bold/ExtraBold) for clear, impactful statements.
  - **Body**: **Inter** (Regular/Medium) for high readability.
  - **Accents**: **JetBrains Mono** or **Roboto Mono** for step numbers, technical labels, and data fields.
</idea>

<idea>
  **Design Movement**: Corporate Futurism
  **Core Principles**:
  1. **Structured Trust**: A grid-based layout that emphasizes stability and reliability, crucial for B2B services.
  2. **Clean Tech**: White space (or "dark space") is used to separate concepts clearly. Less "glow", more "solid".
  3. **Human-AI Hybrid**: Visuals that blend organic curves (voice waves) with rigid geometric shapes (data blocks).
  4. **Efficiency**: The design prioritizes speed of completion with clear, predictable patterns.

  **Color Philosophy**:
  - **Charcoal & Slate**: A softer dark theme using slate grays instead of pure black, feeling more "enterprise software" than "cyberpunk".
  - **Corporate Blue**: A solid, trustworthy blue for primary actions.
  - **Muted Accents**: Secondary colors are desaturated to maintain a professional tone.

  **Layout Paradigm**:
  - **Single Column Focus**: A centered, single-column layout that mimics a high-end document or contract.
  - **Sticky Header/Footer**: Navigation and progress are always visible at the top/bottom.

  **Signature Elements**:
  - **Solid Blocks**: Inputs are solid blocks of slightly lighter gray, clearly defined against the background.
  - **Step Indicators**: A clear, numbered stepper at the top.
  - **Iconography**: crisp, outlined icons for every section header.

  **Interaction Philosophy**:
  - **Standard Form Patterns**: Familiar interactions (tab to next, enter to submit) to minimize friction.
  - **Validation Messages**: Clear, inline validation text.

  **Animation**:
  - **Subtle Fades**: Elements fade in gently. No dramatic movement.
  - **Color Transitions**: Smooth color changes on hover/focus.

  **Typography System**:
  - **Headings**: **Plus Jakarta Sans** for a modern, geometric look.
  - **Body**: **Inter** for standard readability.
</idea>

<idea>
  **Design Movement**: Ethereal Voice Interface
  **Core Principles**:
  1. **Voice Visualization**: The design mimics the visual representation of sound waves—fluid, organic, and dynamic.
  2. **Soft Focus**: Using blurs and gradients to create a dreamy, high-tech atmosphere.
  3. **Conversational Flow**: The form looks like a chat interface or a voice assistant transcript.
  4. **Light in Darkness**: Elements seem to emit their own light against a dark void.

  **Color Philosophy**:
  - **Deep Indigo/Violet Background**: A rich, dark purple/blue base.
  - **Gradient Accents**: Blue-to-Green and Blue-to-Purple gradients for buttons and active states.
  - **White Glow**: Text and key elements have a subtle outer glow.

  **Layout Paradigm**:
  - **Chat-Style Verticality**: Questions appear one by one, stacking upwards like a message history.
  - **Bottom Input**: The active input area is anchored at the bottom, like a chat box.

  **Signature Elements**:
  - **Waveform Visuals**: Animated sound waves in the background or as progress indicators.
  - **Bubble Inputs**: Input fields look like chat bubbles.
  - **Avatar Presence**: An "AI Agent" avatar that "asks" the questions.

  **Interaction Philosophy**:
  - **Conversational**: The user "replies" to the AI.
  - **Auto-Advance**: The next question appears automatically after answering.

  **Animation**:
  - **Typing Indicators**: Simulated typing dots before a new question appears.
  - **Bubble Pop-in**: Chat bubbles spring into view.

  **Typography System**:
  - **Headings**: **Outfit** or **Space Grotesk** for a unique, tech-friendly character.
  - **Body**: **DM Sans** for a friendly, approachable feel.
</idea>
